# filter_rules.md — 文章筛选规则

> 用于 fetch_articles.py / add_article.py 自动预筛，以及 raw/ → knowledge/ 处理时的人工复核。
> 规则来源：用户 2026-07-18 对30篇文章的人工筛选决策。

---

## 排除规则（命中任一即排除）

### 1. 产品发布 / 推广文
- 标志：文章核心是介绍/推广某个具体产品或功能上线，缺乏可迁移的方法论
- 标题特征：`发布`、`上线`、`推出`、`全新`、`凭什么`、`带你速通`、`你真的需要`
- 典型案例：
  - 腾讯混元Hy3发布：Agent能力和产品体验跃升
  - 从0到1带你速通 Marvis马维斯
  - 大圆实测：你真的需要一个长在企业微信中的AI
  - 上线后差点"挤爆"服务器，Marvis凭什么让用户追着6只小马"上班"？

### 2. 产品系列标记
- 标志：标题中含特定产品名，且文章视角是介绍/展示而非方法论
- 产品黑名单：`Marvis`、`马维斯`（Marvis专属推广系列）
- 注意：含混元但侧重技术分析（如推理优化原理）的文章可保留，需人工判断
- 典型案例：
  - 腾讯混元AI Infra如何优化Hy3 Preview（偏产品推广，排除）
  - 腾讯混元Hy3发布（产品发布稿，排除）

### 3. 招募 / 活动公告
- 标志：文章目的是招募人员或宣传活动，无技术内容
- 标题特征：`招募`、`开启`、`计划`、`报名`、`犀牛鸟`
- 典型案例：
  - 2026腾讯犀牛鸟开源人才培养计划正式开启

### 4. 娱乐 / 八卦内容
- 标志：与AI工程实践无关，纯娱乐性
- 典型案例：
  - 鹅厂员工当年高考都考了多少分？

### 5. 新闻稿 / 获奖公告
- 标志：通知性内容，无可提炼的方法论或技术决策
- 标题特征：`获奖`、`突破`、`重要成果`、`杰出论文`、`联合XXX`
- 典型案例：
  - 腾讯PCG联合深圳河套学院：在全双工语音大模型领域取得重要突破，获 ACL 2026 杰出论文奖

### 6. 纯基础设施 / 与AI工程实践关联度低
- 标志：偏向底层平台/运维，非AI工程师日常工作范畴
- 典型案例：
  - 腾讯Ray团队实践：K8s + Ray如何支撑超大规模AI Workload

---

## 保留规则（以下类型优先保留）

1. **方法论文章**：提出或深化一套可复用的工作方法（Harness Engineering、Loop Engineering、Skill设计等）
2. **实践案例**：有具体落地过程、决策路径和可迁移经验的工程实践记录
3. **技术分析**：对某个技术问题的深度拆解，有具体结论可参考（Token控制、上下文压缩、推理优化原理等）
4. **工程经验**：个人/团队视角的真实踩坑与收获，有叙事和洞察
