## 核心问题

「从 token 焦虑到系统化治理——这次想分享的不是几个省 token 小技巧，而是一套从心智模型到工程落地的完整方法。」

AI Coding 的 Token 账单往往"不知不觉就上去了"——一个长 session、一次全量代码分析、一次递归压缩，Token 消耗翻几倍。

## 方法

**三层降本策略**（按 ROI 排序）：

1. **会话管理**（零成本）— 任务结束后关 session、小任务不开最强模型、不重复跑已完成的步骤
2. **上下文精简**（低配置成本）— diff 优先而非读全文件、结构化输入而非自然语言描述。一条结构化 prompt 省 80% Token，效果不变
3. **架构优化**（高收益但需投入）— 知识库按需加载、SubAgent 拆分但计费隔离、token 用量监控 + 告警。**只有可度量的 Token 才能被管理**——配 Token 用量看板，设阈值告警

**反直觉发现**：便宜模型 + 干净上下文 > 最强模型 + 脏长上下文。

## 关键引用

> 「只有可度量的 Token 才能被管理。Token 是钱，也是注意力——浪费 Token 不是浪费钱的问题，是浪费模型注意力的问题。」
